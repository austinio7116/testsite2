How to Connect to Amazon APIs

A step-by-step guide to setting up and using AWS and Amazon service APIs

Overview

Amazon provides APIs through Amazon Web Services (AWS) and standalone product APIs. This guide covers how to get started, authenticate, and make your first API calls.

Popular Amazon APIs

  • AWS (S3, EC2, Lambda, etc.)Cloud infrastructure and services
  • Amazon SP-APISelling Partner API for marketplace sellers
  • Product Advertising APIProduct search and affiliate links
  • Alexa Skills KitBuild voice experiences for Alexa

Step 1: Create an AWS Account

  1. Go to aws.amazon.com and click Create an AWS Account.
  2. Provide your email address, choose an account name, and set a password.
  3. Enter your contact and billing information (a credit card is required, but many services have a free tier).
  4. Complete identity verification and select a support plan.
Note: If you only need the Selling Partner API or Product Advertising API, you will also need an Amazon Seller or Associates account respectively.

Step 2: Create an IAM User and Access Keys

Never use your root account credentials for API access. Instead, create a dedicated IAM user.

  1. Sign in to the AWS Management Console and open the IAM service.
  2. Click Users → Create user.
  3. Enter a username (e.g., api-user) and click Next.
  4. Attach policies that grant only the permissions you need (e.g., AmazonS3ReadOnlyAccess).
  5. After creating the user, go to Security credentials → Create access key.
  6. Choose the Application running outside AWS use case.
  7. Download or copy the Access Key ID and Secret Access Key.
Important: Store your secret access key securely. You will not be able to view it again after this step.

Step 3: Configure Your Credentials

Option A: AWS CLI

Install the AWS CLI and run:

aws configure

You will be prompted to enter:

AWS Access Key ID [None]: AKIAIOSFODNN7EXAMPLE
AWS Secret Access Key [None]: wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY
Default region name [None]: us-east-1
Default output format [None]: json

Option B: Environment Variables

export AWS_ACCESS_KEY_ID="AKIAIOSFODNN7EXAMPLE"
export AWS_SECRET_ACCESS_KEY="wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY"
export AWS_DEFAULT_REGION="us-east-1"

Option C: Credentials File

Create or edit ~/.aws/credentials:

[default]
aws_access_key_id = AKIAIOSFODNN7EXAMPLE
aws_secret_access_key = wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY

Step 4: Install an SDK

Amazon provides official SDKs for most popular languages:

Python (boto3)

pip install boto3

JavaScript / Node.js

npm install @aws-sdk/client-s3

Java

<dependency>
    <groupId>software.amazon.awssdk</groupId>
    <artifactId>s3</artifactId>
</dependency>

Go

go get github.com/aws/aws-sdk-go-v2/aws

Step 5: Make Your First API Call

Python — List S3 Buckets

import boto3

s3 = boto3.client("s3")
response = s3.list_buckets()

for bucket in response["Buckets"]:
    print(bucket["Name"])

JavaScript — List S3 Buckets

import { S3Client, ListBucketsCommand } from "@aws-sdk/client-s3";

const client = new S3Client({ region: "us-east-1" });
const response = await client.send(new ListBucketsCommand({}));

response.Buckets.forEach(bucket => {
    console.log(bucket.Name);
});

Using the AWS CLI Directly

aws s3 ls

Connecting to the Selling Partner API (SP-API)

If you sell on Amazon Marketplace, the SP-API lets you manage listings, orders, and fulfillment programmatically.

  1. Register as a developer in Seller Central under Apps & Services → Develop Apps.
  2. Create a new app and note your Client ID and Client Secret.
  3. Create an IAM role and policy with the execute-api:Invoke permission for SP-API.
  4. Use the Login with Amazon (LWA) OAuth flow to obtain an access token:
curl -X POST https://api.amazon.com/auth/o2/token \
  -d "grant_type=refresh_token" \
  -d "refresh_token=YOUR_REFRESH_TOKEN" \
  -d "client_id=YOUR_CLIENT_ID" \
  -d "client_secret=YOUR_CLIENT_SECRET"

Use the returned access token in the x-amz-access-token header for subsequent API calls.

Connecting to the Product Advertising API

The Product Advertising API (PA-API 5.0) lets you search Amazon's product catalog and generate affiliate links.

  1. Sign up for the Amazon Associates Program.
  2. Once approved, go to Tools → Product Advertising API and generate credentials.
  3. Install the SDK or sign requests manually using AWS Signature Version 4.
  4. Make requests to the SearchItems, GetItems, or GetBrowseNodes operations.
# Example: Search for items (Python with paapi5)
from paapi5_python_sdk.api.default_api import DefaultApi
from paapi5_python_sdk.models.search_items_request import SearchItemsRequest

api = DefaultApi(
    access_key="YOUR_ACCESS_KEY",
    secret_key="YOUR_SECRET_KEY",
    host="webservices.amazon.com",
    region="us-east-1"
)

request = SearchItemsRequest(
    partner_tag="your-tag-20",
    partner_type="Associates",
    keywords="python programming",
    search_index="Books",
    item_count=5
)

response = api.search_items(request)

Best Practices

  • Least privilege: Only grant the IAM permissions each application actually needs.
  • Rotate keys regularly: Set up key rotation policies and never hard-code credentials in source code.
  • Use IAM roles instead of access keys when running on AWS infrastructure (EC2, Lambda, ECS).
  • Enable MFA on your AWS root account and privileged IAM users.
  • Monitor usage: Enable AWS CloudTrail to log all API calls for auditing.
  • Handle rate limits: Implement exponential backoff and retry logic for throttled requests.
  • Use environment variables or a secrets manager (e.g., AWS Secrets Manager) to store credentials.

Troubleshooting

Common Errors

  • InvalidClientTokenId — Your access key ID is incorrect or has been deactivated.
  • SignatureDoesNotMatch — Your secret key is wrong or the request signing is malformed.
  • AccessDenied — Your IAM user lacks the required permissions. Check your attached policies.
  • ExpiredToken — Your temporary session token has expired. Request a new one.
  • ThrottlingException — You are sending requests too fast. Add retry logic with exponential backoff.
